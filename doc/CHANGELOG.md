## 填写指南

### 版本号规则
采用语义化版本号：`主版本号。次版本号.修订号`

- **主版本号**: 重大功能更新或不兼容的 API 修改
- **次版本号**: 向后兼容的功能性新增
- **修订号**: 向后兼容的问题修正

### 更新类型说明

#### 新增功能 (✨)
- 全新的功能特性
- 新增的模块或组件
- 扩展的能力

#### 改进优化 (⚡)
- 现有功能的性能提升
- 用户体验优化
- 代码重构和改进

#### Bug 修复 (🐛)
- 错误修复
- 稳定性改进
- 兼容性问题解决

#### 文档更新 (📚)
- 新增文档
- 文档完善
- 注释补充

#### 其他变更 (🔧)
- 依赖更新
- 配置调整
- 工具链升级

### 填写建议

1. **及时性**: 每次提交代码后及时更新
2. **准确性**: 清晰描述变更内容
3. **简洁性**: 语言精炼，重点突出
4. **完整性**: 不遗漏重要变更

### 示例

```markdown
## [0.3.0] - 2026-03-20

### 新增功能 ✨
- 添加软件搜索功能
- 实现下载进度条显示

### 改进优化 ⚡
- 优化网络检测逻辑
- 提升页面加载速度

### Bug 修复 🐛
- 修复 WiFi 检测在某些设备上失败的问题
- 修复数据加载异常导致的崩溃

### 文档更新 📚
- 完善 API 参考文档
- 添加开发者指南
```
