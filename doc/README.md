## ✍️ 文档编写规范

### 命名规范
- 文件名：全大写 + 下划线分隔（如 `PROJECT_DOCUMENTATION.md`）
- 标题：使用 Markdown 标题层级
- 标签：使用 emoji 图标增强可读性

### 格式规范
- 使用 Markdown 语法
- 代码块标注语言类型
- 表格统一使用管道符格式
- 列表使用短横线 `-`

### 内容规范
- 标题清晰，层次分明
- 段落简短，易于阅读
- 示例代码可运行
- 截图配有说明文字
- 重要信息加粗强调

### 更新规范
- 文档末尾标注最后更新时间
- 涉及版本的标注版本号
- 重大变更在 CHANGELOG 中记录
